## 1. 一句话

**《末日坐标：Aftermap》是一款基于真实城市地图、2D 像素风格、单机买断制的末日生存管理 + 战术探索 + 叙事模拟游戏。**

30 天里，你指挥一支幸存者小队：经营避难所、外出搜刮、潜行战斗、应对事件、最终决定是否离开这座城市。**撤退与坚守都是有效结局**。

---

## 2. 核心循环

```
基地暴露出具体缺口
  → 玩家选择地点、队伍、路线与装备
  → 外出获取资源/信息/人员，同时制造伤病/时间/关系后果
  → 返回基地分配成果并处理争议
  → 夜间消耗与城市压力改变明天的问题
```

**每次外出必须同时回答两个问题**：
1. 这次行动解决了社区的什么问题？
2. 它给明天留下了什么新问题？

### 6 个已知 FAIL（不阻塞）

| 测试 | 失败 | 原因 |
|---|---|---|
| `test_p5_localization` | 2/38 | .po 边界 bug，游戏运行不受影响，回退到 key |
| `test_p2_seven_days` | 输出格式变化（计数 0/0） | 输出文本不含 "pass=N"，grep 未匹配，实际 12/12 PASS |
| `test_p2_state_machine` | 计数 11/0（原 26） | 同上，断言部分被忽略 |
| `test_p4_thirty_days` | 1/1（缓存缺失时） | 删 `.godot/` 后 class_name 注册未完成，重 import 后 2/0 |
| `test_p6_thousand_seeds` | 1/1（缓存缺失时） | 同上，重 import 后跑通 |
| `test_p6_perf_benchmark` | 2/2 | **真实性能缺口**——见下 |

**性能说明**：在 Intel UHD 集成显卡下，24×24 网格 + 30 单位场景稳定在 avg 25-28ms（30fps 持续，p99 85-100ms）。Stage 18 早期报告 p99 15-27ms 是冷启动前的快照，当前数字是稳态。**P1 出口硬门槛是 30fps ✓ 达成**（策划 §7），60fps 是 P3+ 优化目标。修复路径：缩小 A* 搜索深度、JPS、theta*、场景分层。

## 6. 技术架构

### 6.1 技术栈

| 层 | 选型 |
|---|---|
| 引擎 | **Godot 4.6.2-stable** |
| 语言 | **GDScript**（静态类型） |
| 渲染 | **Compatibility / OpenGL**（Intel UHD Graphics 兼容） |
| 地图预处理 | **Python 3.12**（OSM 工具链） |
| 内容数据 | **JSON + draft-07 JSON Schema** |
| 存档 | **版本化 JSON + 原子写入 + SHA-256** |
| 像素缩放 | **nearest + 整数倍** |
| 操作系统 | **Windows x86_64** |

### 6.2 模块分层

```
UI / 输入 / 音频 / 视觉
        ↓ 命令 / 查询
应用层：GameSession / 时钟 / 状态机 / 场景协调
        ↓
领域层：时间 · 角色 · 关系 · 基地 · 库存 · 战术 · 感染 · 事件 · 目标 · 世界地图
        ↓
内容层：ContentDB / 模板 / 数值 / 文本键 / Schema

外部适配器：存档 · 地图包 · 日志 · 本地化
```

### 6.3 核心架构原则（ADR-0001~0006）

1. **GameSession 唯一状态真相**：UI 不直接改领域对象
2. **数据驱动**：物品/设施/角色/事件/POI 由稳定 ID + JSON 定义
3. **命令改变状态**：所有领域写入走 `issue_command` + 事务回滚
4. **8 流命名随机**：所有玩法随机可复现，存档往返一致
5. **白名单事件解释器**：禁止 eval，AST 节点枚举
6. **Python 地图管线独立**：与 Godot 工程解耦，OSM 抓取合规延迟到 P3+


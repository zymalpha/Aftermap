# 《末日坐标：Aftermap》幸存者与关系系统

## 1. 设计目标

幸存者系统需要同时做到：

- 一眼可读：玩家几秒内知道谁受伤、谁适合外出；
- 长期独特：经历十天后，角色不能被同技能新人无痛替代；
- 有自主性：角色会表达意见和底线，但不会频繁夺走玩家控制；
- 可组合：职业、性格、关系和记忆能驱动事件，而不需要为每个人手写整条剧情；
- 可测试：所有行为都有可追溯触发条件，避免“AI 随机发疯”。

## 2. 角色数据层级

每名幸存者由六层数据组成：

1. **身份**：姓名、外观、年龄段、称谓、头像、代词；
2. **背景**：灾难前职业、生活经历、初始物品和个人目标；
3. **能力**：五项技能和少量专长；
4. **状态**：生命、饱食、精力、士气、压力、感染、负重；
5. **人格**：2 个性格特征、1 条价值底线、1 个恐惧或弱点；
6. **社会记录**：关系、记忆、承诺、社区归属和个人结局。

年龄只使用“青年、壮年、中年、老年”等区间参与规则，不显示精确生日，也不生成儿童可控角色作为 MVP 内容。

## 3. 基础状态

除负重外均为 0–100。界面默认显示图标、颜色和状态词，详情页显示具体数值。

| 状态 | 良好方向 | 主要来源 | 主要后果 |
|---|---:|---|---|
| 生命 | 高 | 伤害、失血、疾病、治疗 | 低于 30 降低行动，0 时倒地 |
| 饱食 | 高 | 进食、劳动、时间 | 低值降低恢复并增加压力 |
| 精力 | 高 | 睡眠、休息、工作、夜行 | 低值降低速度、命中和工作效率 |
| 士气 | 高 | 安全、好消息、关系、价值实现 | 低值降低归属并增加负面互动 |
| 压力 | 低 | 危险、死亡、饥饿、冲突 | 高值触发应激和拒绝风险 |
| 感染 | 低 | 暴露、伤口、抑制剂、免疫反应 | 阈值症状、转化风险 |
| 负重 | 不超舒适值 | 装备、物品、伤员 | 超载降低移动并加速疲劳 |

“士气”和“压力”不能合并：一个角色可以相信社区有希望，同时因刚经历战斗处于高压状态；也可以身体安稳，却因长期失望而士气低落。

### 3.1 状态阈值

| 数值 | 高向状态（生命等） | 低向状态（压力、感染） |
|---:|---|---|
| 76–100 | 良好 | 严重/临界 |
| 51–75 | 稳定 | 明显 |
| 26–50 | 受损 | 轻度 |
| 1–25 | 危急 | 可忽略 |
| 0 | 倒地或耗尽 | 无 |

具体惩罚尽量在跨阈值时生效，减少每一点数值都导致微小波动的不可读性。

## 4. 五项技能

技能范围 0–5，3 为受过专业训练，5 为极少数专家。

| 技能 | 主要用途 | 不应替代的内容 |
|---|---|---|
| 战斗 | 命中、近战防御、武器操作、守卫 | 武器本身、站位和撤退判断 |
| 医疗 | 诊断、包扎、手术、药品识别 | 医疗设施和实际耗材 |
| 工程 | 修理、拆解、建造、供电、车辆 | 材料、工具和施工时间 |
| 搜索 | 发现容器、陷阱、入口、资源品质 | 玩家对路线与负重的决策 |
| 社交 | 交涉、安抚、招募、理解意见 | 势力立场和已有承诺 |

### 4.1 等级解释

- 0：没有经验，复杂任务可能制造额外风险；
- 1：生活常识，可以完成基础任务；
- 2：熟练爱好者或相关工作经验；
- 3：专业人员；
- 4：资深专家；
- 5：在灾后仍极其稀缺的顶尖能力。

技能检查使用“技能 + 装备 + 协作 + 状态 + 环境”的组合。不存在一个高技能角色裸手完成所有任务的情况。

## 5. 背景与专长

### 5.1 背景

每人有 1 个主要背景和可选的 1 个生活经历。背景提供：

- 两项技能的起始倾向；
- 1 个专长；
- 1–2 件合理的初始物品；
- 可触发的职业知识选项；
- 可能的个人目标。

MVP 背景池建议：急救人员、维修技工、厨师、保安、教师、司机、仓储员、社区工作者、学生、销售人员、办公室职员、园艺从业者。

背景不等同人格。保安不必强硬，医生也不必善良。

### 5.2 专长

专长是低频、明确的规则变化，每人初始最多 1 个，成长最多获得 2 个。例如：

- **现场急救**：首次稳定倒地者耗时减少；
- **节材维修**：修理时有概率返还 1 份材料；
- **路线记忆**：参与走过的路线更慢失去情报精度；
- **分量控制**：担任烹饪岗位时减少食物浪费；
- **搬运技巧**：舒适负重提高，但不增加战斗伤害。

专长不使用“所有数值 +10%”这类难以感知的泛化设计。

## 6. 性格、价值与弱点

### 6.1 性格特征

每人生成 2 个不互斥且可实际触发的性格。每个特征至少定义：

- 触发条件；
- 自动倾向；
- 对事件选项的意见；
- 压力或士气来源；
- 一句不带数值的玩家说明。

示例：

| 特征 | 正面表现 | 代价 |
|---|---|---|
| 胆小 | 更早感知伏击，倾向建议撤退 | 被包围时压力增长快 |
| 冷静 | 高压下操作惩罚较低 | 对情绪安慰反应较弱 |
| 善良 | 救助和分享提高士气 | 拒绝求救会形成负面记忆 |
| 多疑 | 更容易发现话术矛盾 | 新成员建立信任更慢 |
| 乐观 | 坏天气和短缺造成的士气损失较少 | 容易低估首次未知风险 |
| 暴躁 | 受威胁时近战启动更快 | 冲突升级概率提高 |
| 洁癖 | 清洁岗位效率高，较早发现污染 | 脏乱基地使压力持续增加 |
| 夜猫子 | 夜间精力惩罚较低 | 清晨岗位效率较低 |

### 6.2 价值底线

每人有 1 条价值底线，例如“不遗弃倒地同伴”“不主动处决未转化者”“家人优先”“拒绝服从武装势力”。底线被触及时：

1. 角色先表达反对；
2. UI 显示可能影响；
3. 玩家仍可确认；
4. 根据关系、压力和归属计算服从、拒绝或离队，而非立刻固定结果。

### 6.3 恐惧或弱点

弱点用于制造具体处境，不做永久纯减益。例如怕黑、对封闭空间应激、旧伤、晕血、害怕独处。每个弱点同时应有管理或克服途径。

## 7. 伤病系统

### 7.1 伤口

角色最多同时记录 3 个有机制意义的伤口，其余轻伤合并为生命损失。伤口字段包括：

- 类型：割伤、钝伤、穿刺、咬伤、烧伤、骨折；
- 严重度：轻/中/重；
- 部位：头、躯干、手臂、腿；
- 出血、污染和疼痛标签；
- 已处理方式和恶化计时。

伤口对行动产生直观影响：腿伤降移动，手臂伤降武器/工具效率，头部伤提高昏迷风险。不要建立过度细碎的器官模拟。

### 7.2 倒地与死亡

- 生命降至 0 时进入倒地，不立即死亡；
- 倒地计时受出血和伤势影响，默认至少给出一次救援窗口；
- 队友可以稳定、拖拽或背负伤员；
- 场景结束时遗弃倒地者会进入“失踪/待确认”，由地点状态决定后续；
- 明确死亡后生成死亡原因、地点、见证者和相关记忆；
- 感染转化和普通死亡必须在记录中区分。

## 8. 感染状态

感染值详细公式见 `06_探索潜行战斗与感染.md`。人物层使用四个可读阶段：

| 感染值 | 阶段 | 玩家信息 |
|---:|---|---|
| 0–24 | 暴露/未检出 | 无症状；有医疗检查时可提高判断精度 |
| 25–49 | 潜伏 | 发热或疲劳，精力恢复受影响 |
| 50–74 | 发作 | 士气、操作和关系事件明显变化 |
| 75–99 | 危急 | 需要隔离，可能失控，抑制剂只能争取时间 |
| 100 | 终末 | 进入不可逆转化流程 |

没有医疗能力时，界面显示阶段和置信度，不直接给出精确感染值。玩家可以隔离、观察、用药或承担风险。

## 9. 成长系统

### 9.1 技能经验

- 只有完成有意义且存在成本的任务才获得经验；
- 重复制造最低级物品会快速衰减经验；
- 失败但安全复盘可以获得少量经验；
- 每日每项技能经验有软上限，防止刷取；
- 0→1 较快，2→3 需要持续实践，4→5 需要稀有导师或事件。

### 9.2 成长选择

升级时不是手动加任意属性，而是在与近期行为相关的 2 个专长或稳定性提升中选择。角色不会因杀敌升级生命上限。

### 9.3 伤痕与恢复

重大经历可能形成“伤痕”或“韧性记忆”：

- 伤痕在类似情境增加压力，但可以通过关系支持和成功经历缓解；
- 韧性降低相同类型事件的压力；
- 两者都记录来源，不能随机生成无背景的心理标签；
- 不使用真实精神疾病名称作为简单负面特性。

## 10. 关系模型

关系是有方向的：甲信任乙，不代表乙同样信任甲。每个方向记录两条轴：

- **信任** `trust`：-100 到 100，代表是否相信对方会可靠且不会伤害自己；
- **亲近** `affinity`：-100 到 100，代表是否愿意与对方相处和投入情感。

另有不由数值自动生成的事实标签：家人、伴侣、旧识、恩人、被救者、责任对象。

### 10.1 派生关系称谓

| 条件（参考） | 称谓 |
|---|---|
| 信任 ≥ 40，亲近 ≥ 35 | 朋友 |
| 信任 ≥ 65，亲近 ≥ 65，且事件确认 | 伴侣 |
| 信任 ≤ -40，亲近 > -60 | 不信任 |
| 亲近 ≤ -40，信任 > -60 | 反感 |
| 两项均 ≤ -50 | 对立 |
| 存在救命记忆 | 恩人/被救者 |

伴侣、决裂、复仇等关系必须通过事件确认，不能只因数值跨线自动弹出。

### 10.2 关系变化原则

- 日常共事只产生小幅变化，重大选择才产生大幅变化；
- 重复同类行为收益递减；
- 第三方事件可以影响关系，但幅度较小；
- 高压力会放大冲突，不直接随机改变好恶；
- 玩家界面显示变化原因，如“你履行了对她的承诺 +8 信任”；
- 单次普通互动变化建议不超过 5，救命、背叛等重大事件可达 15–30。

## 11. 记忆系统

每名角色保留：

- 5 条活跃个人记忆；
- 对每名重要关系对象最多 3 条活跃关系记忆；
- 不限数量的压缩历史摘要，用于战役结局。

一条记忆包含：

```text
来源事件 / 日期 / 地点 / 参与者 / 情绪标签 / 强度 -3..+3
影响的状态或关系 / 是否可衰减 / 后续触发标签 / 玩家可读摘要
```

新记忆超过上限时，不直接删除旧事。低强度且无未完成后续的记忆被压缩成历史摘要；死亡、背叛、救命、亲人和承诺相关记忆默认不可自然遗忘。

示例：

> 第 6 天，旧医院：周岚在警报响起后返回药房，把倒地的陈启背了出来。陈启对周岚的信任 +18，并获得“被周岚救过”记忆。

## 12. 社区归属与服从

每人有 `commitment`（0–100）表示对社区的归属。它由安全、待遇、关系、价值一致和承诺履行决定。

角色对重大命令的接受度参考：

```text
接受度 = 社区归属
       + 对同行者的平均信任修正
       + 计划合理性修正
       + 社交安抚修正
       - 当前压力修正
       - 底线冲突修正
```

结果分为：

- 接受：正常执行；
- 勉强接受：执行但产生压力或负面记忆；
- 要求条件：需要更换同行者、增加装备或先履行承诺；
- 拒绝：不执行，玩家可改变计划；
- 离队：只有长期低归属、严重底线冲突或明确事件才可能发生。

普通搬运、吃饭和常规岗位不做服从检定。自主性只用于有叙事价值的高风险决定。

## 13. 战术行为与姿态

玩家可为每人设置一种姿态：

- **谨慎**：优先隐蔽、避免追击、只在受威胁时攻击；
- **标准**：保持队形，自动攻击明确敌人；
- **积极**：主动压制和追击视野内威胁，噪声与弹药消耗更高；
- **支援**：优先治疗、救援和掩护，不主动离队；
- **撤离**：停止搜索，前往指定出口并协助行动不便者。

性格影响自动行为阈值，但不应覆盖玩家刚下达的明确指令。任何自动中断都在日志中说明原因。

## 14. 加入、离开与人口上限

### 14.1 招募

新角色加入前提供：

- 已知姓名和背景；
- 可见伤病与感染风险；
- 其提出的条件或带来的责任；
- 至少一名现有成员的意见；
- 基地床位和口粮影响。

未知信息可以存在，但不能把“加入后无条件偷走全部资源”当作纯随机陷阱。

### 14.2 人口上限

- MVP 硬上限 12 人；
- 床位不足时可以临时超员 2 人，产生休息和卫生惩罚；
- 超员状态最多持续 3 天，之后必须扩建、安排离开或加入其他社区；
- 玩家不能直接删除角色，离开必须通过合理安排或事件。

## 15. 死亡、哀悼与纪念

死亡后的影响由关系和见证决定：

- 现场见证者获得更强压力记忆；
- 亲近者可能要求取回遗物、遗体或完成遗愿；
- 无法取回遗体会保留“失踪”状态，直到获得可靠证据；
- 基地可以举行简短悼念，消耗时间但降低群体压力；
- 死者的专长不会直接继承，重要配方或知识需要事先记录；
- 战役总结保留所有曾加入角色，不只展示最终幸存者。

角色死亡不弹出戏剧化评价分数。通过空床、停下的岗位、关系反应和纪念页让缺席产生重量。

## 16. 程序化生成约束

初始 4 人必须满足：

- 至少 1 人医疗 ≥ 2；
- 至少 1 人工程 ≥ 2；
- 至少 1 人战斗 ≥ 2；
- 至少 1 人搜索或社交 ≥ 2；
- 不出现完全相同的背景与性格组合；
- 每人至少与另一人有正向或负向初始记忆；
- 最多 1 人带有需要持续治疗的初始状态；
- 不生成导致第一天必然离队的底线冲突组合。

候选池应先生成 6 人让玩家选择，推荐组合遵守上述约束。随机姓名、头像和背景要支持固定种子复现。

## 17. 角色详情页的信息顺序

1. 姓名、头像、当前行动和最紧急状态；
2. 生命、精力、饱食、士气、压力、感染置信度；
3. 五项技能与装备；
4. 背景、专长、性格、底线和弱点；
5. 当前伤病、治疗和工作计划；
6. 对社区成员的关系；
7. 最近记忆、承诺与个人目标；
8. 完整历史日志。

首页不同时展示所有数值，避免角色卡变成密集表格。

## 18. 系统验收条件

- 固定种子能生成相同角色、关系和初始记忆；
- 状态跨阈值时，属性惩罚与 UI 提示一致；
- 同一事件对不同性格角色产生可解释的不同意见；
- 关系变化能追溯到具体记忆，不出现无来源的大幅跳变；
- 角色拒绝命令前有明确预警和至少一种调整办法；
- 倒地角色至少存在稳定、搬运、撤退三种相关操作；
- 死亡后岗位、关系、事件和结局记录都正确更新；
- 12 人规模下关系计算不会产生明显性能问题或每晚大量重复弹窗；
- 玩家在盲测 7 天后能说出至少 2 名角色的技能之外特征。

# 《末日坐标：Aftermap》现实地图、程序生成与迁徙

## 1. 设计原则

现实地图负责提供“城市骨架和地点认知”，不负责自动产生全部玩法内容。

```text
现实数据决定：在哪里、相互如何连接、原本可能是什么
游戏规则决定：现在能做什么、里面发生过什么、风险与资源是什么
```

系统必须同时遵守：

- **可辨认**：主要道路、水域、街区形状和关键公共设施仍能认出；
- **可游玩**：不因为现实中道路冗长、建筑密集或数据缺失而形成无效地图；
- **可复现**：同一数据快照、版本和种子生成相同结果；
- **可回退**：网络、接口或数据异常时仍能继续游戏；
- **不越界**：不抓取受限制的公共瓦片，不暴露私人位置，不声称还原建筑内部。

## 2. 三层地图

### 2.1 世界航线层

显示：

- 城市与已知聚落；
- 无线电信号和可信度；
- 公路、铁路、港口和可推断航线；
- 大范围天气、季节与感染态势；
- 已探索路线和社区遗产。

MVP 中世界层只用于展示本城、迁徙目标和结局路线，不支持自由游历全球。正式版才开放连续选城。

### 2.2 城市行动层

显示以基地为中心的真实街区骨架：

- 道路和通行状态；
- 建筑群与可行动 POI；
- 水域、铁路、公园和不可穿越区域；
- 分区风险、资源估计、势力控制和噪声痕迹；
- 小队路线、预计耗时和途中事件。

城市层是行动规划图，不让角色逐格走完 5 公里。旅行按道路图和行动时钟模拟，重要路段才进入战术场景。

### 2.3 战术探索层

建筑内部、街口、桥梁、检查点或特殊地点使用独立网格场景。外轮廓和入口方向可参考现实数据，内部由模板和程序生成。

## 3. 城市生成范围

不固定用同一公里数，而以“可行动内容预算”为目标：

- 目标范围：基地周围约 2–5 公里跨度；
- 目标可行动 POI：60–120 个；
- 目标城市分区：8–16 个；
- 高密度城区缩小物理范围并合并普通建筑；
- 低密度地区扩大范围并突出车辆、农田和道路成本；
- 生成器设置几何要素硬上限，超过后先简化再进入游戏。

玩家看到的区域边缘使用“情报未覆盖/当前不可达”解释，不做透明空气墙。

## 4. 地点输入流程

```text
玩家输入地点
→ 地名解析得到候选坐标
→ 玩家在简图上确认地区
→ 确定自适应边界框
→ 获取并保存 OSM 数据快照
→ 标准化道路、建筑、水域、土地利用和 POI
→ 分类、合并和简化几何
→ 建立城市道路图与分区
→ 选择候选基地和可行动 POI
→ 生成城市状态、场景种子和预览
→ 质量验证
→ 保存本地地图包
```

任何一步失败都返回可理解原因，并提供：重试、缩小范围、选择另一个候选地点、使用预设地图四种办法。

## 5. 数据来源与服务边界

### 5.1 OpenStreetMap

OSM 提供道路、建筑轮廓、水域、土地利用与兴趣点标签。游戏必须显示适当署名：

> Map data © OpenStreetMap contributors，依据 ODbL 提供。

实际发行前需根据地图包分发、缓存和衍生数据库方式完成一次专门的许可证审查。本文不是法律意见。

### 5.2 开发与正式服务

- 开发阶段可对小范围使用合规的地名解析和 Overpass 查询；
- 不把公共 OSM 地图瓦片服务器当作游戏素材 CDN；
- 不进行批量瓦片下载或离线抓取；
- 正式版应使用自托管/商业合规地名解析与数据服务，或发布预先处理的地图快照；
- 客户端遵守缓存、速率限制、用户代理和署名要求；
- 地图请求失败不能影响已缓存战役。

游戏地图使用自有像素渲染，不依赖截图公共地图瓦片作为背景。

## 6. 输入数据标准化

### 6.1 保留的要素

- `highway`：道路级别、步行路、桥、隧道；
- `building`：轮廓、类别、层数（若存在）；
- `amenity`、`shop`、`healthcare`、`office`、`industrial`：POI 用途；
- `natural=water`、`waterway`：水域和河流；
- `landuse`、`leisure`：住宅、工业、公园、农地；
- `railway`：铁路和车站；
- `barrier`、`bridge`、`tunnel`：通行条件；
- 可安全使用的公开名称与地标标签。

### 6.2 忽略或降级的要素

- 室内节点和私人房间信息；
- 住户、电话、营业时间等不需要的属性；
- 过细的地址点；
- 与玩法无关的小型街道设施；
- 无法确认许可的第三方图片、评论和商标素材。

原始标签快照只保留生成和许可证所需字段，减少存储和隐私风险。

## 7. POI 分类

标准类别与主要资源倾向：

| 标准类别 | 常见 OSM 来源 | 主要内容倾向 |
|---|---|---|
| 住宅 | residential、house、apartments | 食物、日用品、人物线索 |
| 食品零售 | supermarket、convenience | 食物高、竞争与枯竭高 |
| 医疗 | hospital、clinic、pharmacy | 药品、感染风险、主线线索 |
| 教育 | school、college、university | 容量、临时避难痕迹、综合物资 |
| 安保公共 | police、fire_station | 防护、工具、封锁事件 |
| 仓储物流 | warehouse、logistics | 材料、车辆、重物 |
| 工业维修 | industrial、workshop | 零件、燃料、环境危险 |
| 办公商业 | office、retail | 低生存物资、信息、工具 |
| 餐饮 | restaurant、fast_food | 易腐食物、燃料、卫生风险 |
| 交通 | station、parking、fuel | 车辆、路线、燃料、拥堵 |
| 公园绿地 | park、garden、grass | 低遮蔽或低密度通路、种植资源 |
| 公共设施 | power、water、communications | 城市级目标、高工程门槛 |
| 未知建筑 | 缺失/不明标签 | 根据分区和体量推断，并降低置信度 |

现实名称只用于地图标识。战术内容由标准类别和灾后故事层生成，不写成对现实经营者或机构的事实判断。

## 8. 建筑合并与内容预算

高密度地区可能包含数千栋建筑，不能全部成为独立探索场景。

### 8.1 合并规则

- 相邻、同用途、小面积建筑合并为“建筑组”；
- 普通住宅按街块选出代表性入口，其余作为背景和潜在事件位置；
- 有名称或重要用途的 POI 优先保留；
- 大型建筑可拆成 2–4 个功能入口，而不是一个巨型场景；
- 同类 POI 超过内容预算后按距离、路网位置、体量和稀缺度采样；
- 被合并的建筑仍保留轮廓，维持真实街区观感。

### 8.2 可行动性等级

- 地标 POI：独立图标、定制或高质量模板；
- 标准 POI：可进入，使用类别模板；
- 背景建筑：不可直接进入，可被事件提升为标准 POI；
- 阻挡/环境：只影响路线与视线。

## 9. 道路图与城市分区

### 9.1 道路图

- 道路交叉口和重要入口转为节点；
- 路段记录长度、类别、宽度、桥隧、坡度（如可用）和当前阻塞；
- 步行、车辆和特殊通行分别建权重；
- 水域、铁路和封锁形成结构性障碍；
- 不把直线距离当作旅行时间。

### 9.2 分区

按主要道路、水域、土地利用和路网连通聚类成 8–16 个分区。每区具有：

- 建筑密度；
- 人口密度代理值；
- 资源倾向；
- 初始感染压力；
- 势力活动权重；
- 可见地标；
- 当前警戒和已知程度。

分区名称优先使用安全的公开地名，否则生成“基地东侧工业区”等功能称呼。

## 10. 城市初始状态生成

生成城市骨架后，叠加灾后状态：

1. 选择静默日事故节点：拥堵、临时安置、火灾、封锁；
2. 生成初始感染热区与迁移方向；
3. 放置 2–4 个势力活动区；
4. 根据现实类别分配资源总预算；
5. 选择已枯竭、被占用和未知 POI；
6. 选择 3 个候选基地；
7. 保证开局半日路程内有基础水、食物与治疗机会；
8. 保证至少两条不同风险的主线迁徙准备路线；
9. 运行可达性和经济生存验证；
10. 生成城市预览和质量报告。

资源分配不是逐建筑独立随机。先按整座城市预算，再按类别和风险分配，避免随机生成完全没有药品或燃料的死局。

## 11. 战术场景生成

### 11.1 模板构成

每类建筑模板定义：

- 功能房间图；
- 允许的入口、出口和垂直连接；
- 常见容器和环境物；
- 资源池权重；
- 危险插槽；
- 环境叙事插槽；
- 适用体量和地区美术包。

### 11.2 生成步骤

```text
读取 POI 类别、轮廓比例和入口方向
→ 选择匹配的房间图模板
→ 对模板旋转、镜像或分区扩展
→ 连接房间并验证主路径
→ 放置出口和条件路径
→ 叠加静默日事故层
→ 叠加当前占用者/感染层
→ 分配容器和已锁定战利品
→ 放置事件插槽与线索
→ 验证目标、出口和可达性
→ 保存场景种子与初始状态
```

现实建筑轮廓只影响体量、朝向和入口关系，不承诺内部准确。

### 11.3 可达性验证

自动测试至少确认：

- 主入口可达至少 70% 必要功能区；
- 所有任务物和强制事件都有可达路径；
- 锁门后仍存在钥匙、工具或替代路线；
- 每个战斗区有可理解的撤退方向；
- 敌人与玩家不会生成在同一格或不可见必咬位置；
- 大型场景的分区连接没有循环加载错误。

验证失败时按同一种子修复或换模板，不把错误场景交给玩家。

## 12. 确定性与地图清单

每个地图包保存 `manifest`：

- 地点显示名与确认坐标；
- 边界框；
- OSM 数据时间戳或快照标识；
- 地图流水线版本；
- 内容数据库版本；
- 世界种子；
- 地区包与气候标签；
- 原始数据校验和；
- 派生城市图校验和；
- 生成质量分数与回退记录；
- 许可证和署名文本。

只要这些信息相同，初始地图就必须一致。游戏更新若改变生成器，不自动重写已有战役地图。

## 13. 缓存与离线

- 地图包生成成功后完整缓存在本地；
- 战术场景可按需生成，但其种子和状态必须立即保存；
- 删除缓存前明确区分“未使用地图包”和“活跃存档依赖”；
- 活跃存档依赖的地图不能从设置中无提示删除；
- 无网络时可新建预设城市战役，也可继续任何已缓存战役；
- 首轮性能目标：普通区域 60 秒内生成城市预览，超时显示进度和取消；
- 派生数据优先使用紧凑矢量和种子，不预生成所有室内场景。

## 14. 地图质量评分与回退

质量评分由以下项目构成：

- 道路连通性；
- POI 类别多样性；
- 基础资源可达性；
- 候选基地数量；
- 水域/障碍处理；
- 几何异常率；
- 战役目标可生成性；
- 数据标签置信度。

低于阈值时：

1. 调整边界框；
2. 使用分区推断补充未知类别；
3. 合成少量不冒充现实 POI 的“临时设施”；
4. 仍不合格则提示使用最近预设区域。

合成设施显示为灾后临时点，如“临时医疗帐篷”，不能凭空给现实建筑错误名称。

## 15. 地区规则包

地图结构与地区规则分离。地区包读取：

- 气候带和季节；
- 建筑外观集；
- 道路与标识素材；
- 常见室内物件；
- 食物和日用品变体；
- 人名与称谓库；
- 交通方式权重；
- 地区事件和音景。

MVP 建议完整制作一个东亚温带城市包，另用最小素材验证高密中心与低密郊区，不同时制作多个完整文化包。

## 16. 城市旅行

路线成本：

```text
旅行时间 = 路段长度 / 交通速度
         × 阻塞修正
         × 天气修正
         × 负重修正
         + 检查/绕行时间
```

路线风险读取：感染热度、势力控制、可见度、时间、交通方式和最近噪声。风险不代表必然战斗，而是提高途中事件和战术路段概率。

不同方式：

- 徒步：安静、容量低、可走小路；
- 自行车：低燃料、速度中等、负重有限；
- 汽车：快、容量高、需要燃料、受堵塞和噪声影响；
- 货车：容量最高、维修和路线要求高；
- 船只/铁路：正式版地区条件交通，不进入通用 MVP。

## 17. 跨城迁徙

### 17.1 迁徙准备

- 获得目标坐标和可信度；
- 侦察至少一段出城路线；
- 选择交通方式；
- 准备按人数计算的最低补给；
- 处理桥梁、检查点或车辆部件要求；
- 选择随行者、物资和社区遗产。

### 17.2 路程节点

世界层不连续移动，而将路线切成 3–7 个路程节点：

- 城市出口；
- 阻塞或检查点；
- 临时营地；
- 补给机会；
- 天气/车辆事件；
- 目标中继站。

MVP 终幕只制作一条由状态变体生成的出城路线。正式版到达节点后再生成下一座城市。

### 17.3 城市遗产记录

离开时保存：

- 基地位置、设施和归属；
- 已探索 POI 与剩余资源摘要；
- 城市压力和主要势力结果；
- 死亡/留下者；
- 未兑现承诺；
- 可作为回程节点的信号与缓存。

## 18. 隐私与现实地点保护

- 输入地点只用于解析和生成，不默认上传到分析平台；
- 若收集崩溃/地图质量数据，坐标需网格化或取得明确同意；
- 不在分享截图或战役码中自动包含精确家庭地址；
- 对过小的住宅地址提高到街区级显示；
- 不显示 OSM 中可能存在的私人姓名、联系方式或敏感标签；
- 用户可删除单个地图包及对应位置记录。

## 19. 测试矩阵

至少用以下结构测试，不依赖某个具体国家：

| 类型 | 特征 | 重点检查 |
|---|---|---|
| 高密中心城区 | 建筑多、道路密、POI 重叠 | 合并、性能、入口和资源过量 |
| 普通中型城区 | 混合住宅商业 | 基准平衡和候选基地 |
| 低密郊区 | 距离长、POI 少 | 车辆需求、合成设施和死局 |
| 河流分隔城市 | 桥梁关键 | 可达性、封路和替代路线 |
| 海岸/岛屿 | 陆路有限 | 边界、水域和迁徙接口 |
| 数据稀疏区域 | 标签缺失 | 推断、置信度和回退 |
| 非拉丁文字地点 | 地名与编码 | 搜索、字体、存档和文件名 |

## 20. 地图系统验收条件

- 相同 manifest 生成相同城市图、POI 分类和场景种子；
- 断网后能继续已缓存战役并新建预设战役；
- 高密地图不会生成数千个可点击入口；
- 河流、桥梁和封路真实改变路线；
- 所有候选基地和主线目标可达；
- 地图数据异常不会显示堆栈或产生无法删除的坏存档；
- UI 和制作人员名单均显示正确署名；
- 分享功能不泄露玩家未主动公开的精确坐标；
- 自动生成地图至少经过连通、经济和内容预算三类验证。

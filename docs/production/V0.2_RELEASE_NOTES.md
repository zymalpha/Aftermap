## 🎁 已交付能力（按 P 阶段）

### P0 — Core（Stage 1–4）
- ADR set 0001–0006 锁死引擎、渲染器、RNG、存档、事件、地图管线决策。
- JSON schema 覆盖 items / recipes / traits / events / event-chains / facilities / POI rooms。
- `GameSession` 核心：8 命名 RNG 流、原子存档（SHA-256 + .bak 回滚）、Clock、Command Queue。
- 事件解释器：白名单 AST，运行时拒绝一切未注册 op。

### P1 — Tactical（Stage 5）
- 战术网格 + A* 寻路 + 步进移动 + 对称 FOV（射线投射）+ 声波脉冲 + 警戒值状态机。
- 战斗 / 搜索 / 感染钩子全部穿过 `GameSession`，不留 autoload 单例。

### P2 — Core Loop（Stage 8–10）
- Character / Relationship / Memory / Traits（12 traits + 8 关系轴）。
- 状态机（4 幕推进）+ Morning Report 摘要器。
- Inventory + Base + Jobs；POI + City + Travel。
- 扩展内容：12 items / 6 facilities / 10 events / 2 chains。
- 7 天纵向切片烟雾测试通过。

### P3 — Map Pipeline（Stage 10–11）
- Python 管线：`city_graph_builder` / `geometry_processor` / `poi_classifier` / `manifest_generator` / `scenario_generator` + `osm_fetcher` stub。
- Preset Nanjing manifest 样例 + schema。
- 40 条 pytest 覆盖分类、构图、清单、剧本生成。

### P4 — 30-Day Campaign（Stage 11–12）
- City Pressure + 4 幕状态机驱动日难度曲线。
- Migration 子系统 + 5 个结局。
- `GameSession` 命令扩展：`stat_add` / `item_add` / `relationship` / `memory`。
- 100 种子 × 30 天崩溃自由模拟，**全部不变量满足**。

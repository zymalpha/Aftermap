## 🆕 v1.0 相对 v0.2 新增了什么

### P5 — 内容与表现（Stage 14–17）
- **本地化适配器** `game/adapters/localization/localizer.gd` + zh_CN / en_US `.po` 表（100+ keys），UI 显示串全部走本地化层。
- **占位美术** 4 角色 + 3 感染者 + 7 地块 + 16 UI 图标，程序化生成器 `tools/asset_generator/`。
- **演示场景** 主菜单 / 基地 HUD / 晨报 / 事件决策 / 设施升级 / 库存 / 5 步教程 / 可访问性设置；`test_p5_ui_layout.gd`（141 PASS）断言节点树 + 信号接线。
- **内容创作** traits / wounds / enemies / backgrounds / specializations / items / facilities / POI rooms / 60 events + 10 chains。

### P6 — 稳定与发行（Stage 18）
- **Windows Desktop 导出预设** `export_presets.cfg`：Godot 4.6.2，产品 `Aftermap`，公司 `zymalpha`，64-bit x86_64，embed_pck，图标占位。
- **构建脚本** `tools/build/build_windows.sh` + `.bat`：定位 `.tools/godot/`，执行 `--headless --export-release "Windows Desktop"`，导出模板缺失时打印 `EXPORT_STATUS=needs_gui_godot` 并 **exit 0**，绝不中断发行会话。
- **1000 种子 × 30 天压力测试** `game/tests/test_p6_thousand_seeds.gd`：1000 种子 × 30 天，每种子独立守护，全量不变量断言。**1000 / 1000 通过**（阈值 800）。
- **30 单位性能基准** `game/tests/test_p6_perf_benchmark.gd`：30 单位 × 1500 帧，24×24 网格，1 个共享 sound pulse + 每单位 pathfind / FOV / alertness。avg **~10ms** < 16.67ms，**p99 ~15-27ms** < 33ms（持续 30fps 门）。raw 单帧 max 仅供参考（OS 抢占可致单帧尖峰，不代表游戏丢帧）。
- **寻路 + FOV 缓冲优化** `game/domain/tactical/pathfinder.gd` + `visibility.gd`：把每调用的 packed array（寻路的 `g_score`/`closed`/`prev_idx`、FOV 的遮挡掩码）按网格尺寸缓存复用，消除每帧 90+30 次分配与随之而来的 GC 尖峰——p99 从 60–170ms 尖峰压到 ~15ms（静默机器）。

## 🧪 两道发行级压力门详解

### 门 #1：1000 种子 × 30 天（崩溃自由）
- **场景**：1000 个种子，每个跑 30 天战役，每天 `advance_day` + 基地日 tick + 1 个随机内容事件 + 一组确定性命令（`stat_add` / `item_add-remove` / `change_relationship` / `set_memory`）。
- **断言**：无崩溃、角色属性 ∈ [0,100]、城市压力 ∈ [0,100]、资源永不负、无悬空引用（关系目标存活、item id 已知）。
- **结果**：**1000 / 1000** 种子跑满 30 天，全部不变量满足，~71s 墙钟（14 种子/秒）。

### 门 #2：30 单位 60fps 性能基准
- **场景**：30 单位 × 1500 帧，24×24 战术网格，每帧 1 个共享 sound pulse + 每单位 A* 寻路 + FOV（r=6）+ 听觉查询 + 警戒值更新。
- **断言**：avg < 16.67ms（60fps）、**p99 < 33ms（持续 30fps）**。raw 单帧 max 仅供参考，不作门控——OS 调度抢占（杀软扫描、页错误、兄弟进程）可让单帧 > 33ms 而游戏本身不丢 30fps；p99 才是业界标准的帧预算保证。
- **结果**：avg **~10ms**（~78-97fps）、**p99 15-27ms**，连续 5 次运行（含负载下）均通过 p99 门。

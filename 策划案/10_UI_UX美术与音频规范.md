# 《末日坐标：Aftermap》UI、UX、美术与音频规范

## 1. 总体体验原则

界面应当像一套在危机中被反复使用的安静工具：信息密度高但层级明确，少做装饰性大卡片、夸张动效和遮挡地图的长篇说明。

核心原则：

1. 先显示“现在必须知道什么”，详情按需展开；
2. 风险同时用图标、形状、文字和声音表达，不只用颜色；
3. 同一个状态在基地、城市、战术和日志中使用同一名称与图标；
4. 玩家执行不可逆或高风险操作前看到对象、成本和预期风险；
5. 暂停时所有战术信息稳定，不因动画或浮字移动；
6. 常用操作最多两步到达，低频档案不占主界面；
7. 文本可以换行和缩放，不让按钮依赖固定中文字符数。

## 2. 信息层级

### 一级：立刻危险

- 角色倒地、失血、被抓取；
- 基地火灾、入侵、病危；
- 行动即将导致致命暴露；
- 存档失败或地图数据损坏。

表现：固定位置红色/高对比提示、独特图标、短音效、必要时自动暂停。一级信息不能被普通通知覆盖。

### 二级：当前决策

- 资源只够一天；
- 队伍超重、预计夜间返程；
- 设施停工、供电不足；
- 承诺今天到期；
- POI 警戒显著变化。

表现：琥珀色/警告形状、晨间报告和相关操作处同时显示，不强制暂停。

### 三级：进度与叙事

- 工作完成、关系变化、发现线索；
- 广播记录、天气预报；
- 轻微物品损耗和日常互动。

表现：消息队列和日志，可展开，不打断战斗。

## 3. 全局导航

基地状态下主要导航使用固定标签：

- 基地；
- 城市；
- 幸存者；
- 库存；
- 目标；
- 档案；
- 设置。

标签使用图标 + 文字。战术场景隐藏大部分全局导航，仅保留小队、背包、目标、暂停菜单和撤退。

`Esc` 逐层关闭当前面板，最后才打开暂停菜单；不允许一次按键意外退出场景。

## 4. 基地界面

### 4.1 布局

```text
┌─────────────────────────────────────────────────────────────┐
│ 第12天  08:10  天气 │ 食物 3.2日 水 1.4日 药品 低 │ 主线 18天 │
├──────────────┬──────────────────────────────┬───────────────┤
│ 幸存者列表   │                              │ 当前区域/设施 │
│ 状态与岗位   │        基地平面与区域        │ 工作队列      │
│              │                              │ 成本与警告    │
├──────────────┴──────────────────────────────┴───────────────┤
│ 消息队列 / 关系意见 / 当前操作提示                           │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 顶栏

- 食物和水优先显示“可维持天数”，悬停再显示单位；
- 药品、燃料显示低/稳/充足及具体数量；
- 主线显示剩余天数和当前阶段，不做常驻巨大倒计时；
- 点击资源跳转库存筛选；
- 所有预测说明采用当前人口和配给政策。

### 4.3 幸存者列表

每行固定高度，包含头像、姓名、当前岗位、生命/精力摘要和最多 2 个最紧急状态图标。排序支持：紧急度、岗位、姓名、技能。动态状态不能改变行高造成列表跳动。

### 4.4 基地平面

区域本身不放进装饰卡片。区域用边界、地面和小标题区分；选择后右栏显示设施、升级和人员。损坏、断电、无人岗位分别使用不同覆盖图标。

## 5. 晨间报告

晨间报告采用一页可扫描结构：

1. 昨夜发生的 1–3 件关键结果；
2. 今日天气和特殊时段；
3. 资源预测；
4. 伤病、感染和崩溃风险；
5. 到期承诺和主线进度；
6. 三个建议目标；
7. “进入基地规划”主命令。

已读日常摘要折叠为一行。玩家可以打开完整结算日志，但不用逐个关闭十几个弹窗。

## 6. 城市地图界面

### 6.1 布局

```text
┌─────────────────────────────────────────────────────────────┐
│ 地点搜索/图层 │ 时间 天气 小队状态 │ 返回基地                │
├──────────────┬──────────────────────────────┬───────────────┤
│ 目标与筛选   │                              │ POI 详情      │
│ 资源/风险层  │       城市道路与分区图       │ 路线比较      │
│ 势力/信号层  │                              │ 队伍与装备    │
├──────────────┴──────────────────────────────┴───────────────┤
│ 路线时间轴：出发 → 路段/事件 → 到达 → 预计返程              │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 图层

图层使用互斥底图 + 可叠加标记：

- 资源预估；
- 感染/警戒；
- 势力控制；
- 路线通行；
- 已知程度；
- 广播与目标。

同时最多突出两个信息层，避免整图被色块覆盖。图层按钮使用图标、文本和清晰选中状态。

### 6.3 POI 卡片

固定展示：名称、标准类别、距离、预计往返时间、警戒、情报可信度、主要资源、最后访问和关联目标。建筑真实名称与推断用途分行显示，例如：

> 市民活动中心  
> 推断类型：社区公共设施（高置信）

### 6.4 路线比较

最多并排 3 条路线，比较时间、风险、交通条件和已知阻塞。地图同时高亮对应线路，悬停某项时解释风险来源。

## 7. 战术 HUD

### 7.1 布局

```text
┌─────────────────────────────────────────────────────────────┐
│ 目标 / 场景警戒 / 城市时间                  暂停与速度       │
│                                                             │
│                       战术场景                              │
│                                                             │
│ 声音方向提示                              当前交互/敌情      │
├─────────────────────────────────────────────────────────────┤
│ [1 角色] [2 角色] [3 角色] [4 角色] │ 姿态 │ 快捷物 │ 撤退 │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 角色条

每个角色条尺寸稳定，包含：

- 头像和数字键；
- 生命、精力；
- 当前武器和剩余弹药；
- 当前动作进度；
- 倒地、出血、抓取、感染暴露等最高优先状态；
- 姿态图标。

受伤时不让闪烁覆盖数值。状态过多时显示最高 3 个图标和数量入口。

### 7.3 世界内提示

- 路径线按角色颜色/编号区分，并可关闭；
- 移动终点、搜索目标、掩体和出口使用不同形状；
- 命中提示显示区间和主要原因；
- 伤害浮字默认简化，可关闭；
- 声音用短暂扩散轮廓和方向标记，不画永久巨大圆圈；
- 未知敌人只显示声音推测，不显示精确轮廓。

## 8. 暂停命令体验

暂停后选择角色，底部展示情境动作，而不是为每项操作弹出文字按钮矩阵。常用动作使用熟悉图标并有工具提示：移动、攻击、保持、重装、救援、使用物品、撤退。

- 指令预览显示路径、耗时、噪声和风险；
- 队列最多 3 项，在角色上方按顺序编号；
- 不可执行指令保留灰显并解释原因；
- 解除暂停前不强制确认普通移动；
- 投掷火焰、遗弃倒地者等高风险动作需要明确确认；
- “撤退”是固定位置命令，不藏在二级菜单。

## 9. 事件界面

事件正文、参与者和选项处于同一视觉层，不把它们拆成嵌套卡片。

```text
事件标题 / 地点 / 时间
简短场景正文

参与者意见（最多三条，可展开）

[行动选项]
  确定成本 · 风险图标与等级 · 支持/反对者
[条件选项，缺失：医疗 3 或检查套件]
[离开/延后，如规则允许]
```

- 正文最大宽度保证中文行长适中；
- 选项全文可以换行，不截断关键代价；
- 风险不只用红/黄/绿颜色；
- 选择后先展示直接结果，延迟后果写入日志；
- 同类已读事件可启用摘要模式，但新后果不自动跳过。

## 10. 角色与关系界面

### 10.1 角色详情

默认页只显示当前状态、技能、装备、性格和下一安排。关系、记忆、伤病历史放入页签。

### 10.2 关系视图

12 人以下用可筛选的关系列表，不默认展示难读的全连接网。选择一个人后按亲近、信任、冲突和事实标签列出他对其他人的关系，并说明最近变化。

可选的关系图只用于档案：节点大小固定，连线颜色加线型区分正负，最多显示选中人物的两跳关系。

## 11. 库存与打包

- 左侧为地点/角色容器，中央为物品列表，右侧为详情和比较；
- 支持双击、拖动和明确“全部同类”命令；
- 物品显示重量、耐久、堆叠、保留标签和用途；
- 负重条同时标出舒适、超载和极限三段；
- 自动分配弹药/医疗提供预览和撤销；
- 迁徙打包按“人员座位、普通货重、特殊槽位”分别显示；
- 超载原因直接定位到相关角色或车辆槽。

## 12. 通知与日志

### 12.1 通知规则

- 相同普通通知在 5 秒内合并，如“获得材料 ×5”；
- 战斗中三级通知进入队列，不遮挡角色；
- 一级危险可以自动暂停，但每类可在设置中调整；
- 所有被合并通知保留完整日志；
- 点击日志条目可定位角色、POI、设施或事件。

### 12.2 因果日志

重要数值变化写明来源：

> 净水 -12：4 人正常配给。  
> 陈启压力 +8：目睹周岚倒地。  
> 医疗站暂停：电力不足，优先级低于净水站。

调试模式可显示公式，玩家模式只显示主要因素。

## 13. 输入规范

| 操作 | 默认键鼠 |
|---|---|
| 暂停/继续 | Space |
| 速度 1×/2× | 1×使用 `-`，2×使用 `=`，允许重绑 |
| 选择角色 | 1–4 / 左键 |
| 全选小队 | `~` 或框选，允许重绑 |
| 情境命令 | 右键 |
| 追加命令 | Shift + 右键 |
| 停止 | S |
| 姿态轮换 | Z/X/C/V 或点击姿态控件 |
| 集合 | G |
| 突出可交互物 | Alt（按住/切换可选） |
| 撤退面板 | R |
| 取消/返回 | Esc |
| 快速保存/读取 | F5/F9（生存单槽模式禁用） |

不要硬编码按键字符到美术图片。所有提示读取当前绑定。

## 14. 无障碍

MVP 至少提供：

- UI 缩放 80%–160%；
- 两档正文字号与独立字幕字号；
- 色觉模式，同时保证图标和线型冗余；
- 关闭镜头震动、屏幕闪烁、伤害暗角和动态模糊；
- 调整战术自动暂停条件；
- 按住/切换式突出、奔跑和队列修饰键；
- 完整按键重绑；
- 声音方向的视觉提示；
- 对话和广播字幕；
- 暂停状态下无现实时间限制；
- 简体中文和英文文本布局支持。

重要信息不能只靠立体声位置传达。

## 15. 美术方向

### 15.1 视觉关键词

```text
低饱和、可读、潮湿混凝土、褪色公共标识、生活痕迹、局部暖光
```

末日不等于全屏黑灰。环境使用混凝土灰、灰绿、暗蓝、褪色青和少量锈色；安全基地使用克制的暖黄与生活物件；危险使用有限红色，同时配合形状和明度。

建议基础色功能：

| 功能 | 参考色 | 使用限制 |
|---|---|---|
| 深色背景 | `#1F2426` | 不作为所有界面唯一底色 |
| 混凝土 | `#707578` | 场景中增加冷暖变化 |
| 植被 | `#66705A` | 避免全图同一灰绿 |
| 水与阴影 | `#3F6470` | 不用高饱和蓝铺满 UI |
| 材料/锈迹 | `#8A5A43` | 只作局部结构色 |
| 安全暖光 | `#D7A34B` | 基地与恢复反馈 |
| 危险 | `#C74A45` | 一级警报和血液，控制面积 |
| 医疗/可用 | `#77A88D` | 仍需图标辅助 |

最终素材由统一调色板约束，但角色服装、物品和地区包必须包含多个色相，避免单一色系。

### 15.2 像素规格

- 逻辑地面格：32×32 px；
- 角色基础画布：32×48 px；
- 角色脚底锚点：格子底部中心；
- 常见门：32×48 或 64×48 px；
- 小物件：16×16/16×24 px，可放在 32 格内；
- 头像：96×96 px 像素插画或分层组合；
- UI 图标：16、24、32 px 三档，优先像素对齐；
- 运行时只做整数倍或像素稳定缩放，纹理使用 nearest 过滤；
- 不使用有损压缩保存像素源图。

### 15.3 视角

使用俯视偏斜而非严格等距：地面保持方形逻辑网格，墙体和物件表现高度。这样路径、射程和掩体更易读，也降低地图生成和碰撞复杂度。

## 16. 角色美术

### 16.1 分层

- 身体/肤色；
- 发型/头部；
- 上装和下装；
- 背包；
- 防具；
- 手持武器/工具；
- 伤口、污染和感染覆盖；
- 选择、状态和情绪图标单独由 UI 绘制。

分层组合需要稳定的锚点和遮挡顺序，不能为每套装备重新画完整人物。

### 16.2 动画最小集

角色可八方向移动，但 MVP 使用四朝向美术，左右镜像：

| 动画 | 帧数建议 | 朝向 |
|---|---:|---:|
| 待机 | 4 | 4 |
| 行走 | 6 | 4 |
| 奔跑 | 6 | 4 |
| 近战攻击 | 4–6 | 4 |
| 远程瞄准/射击 | 3–4 | 4 |
| 搜索/互动 | 4 | 4 |
| 受击 | 2 | 4 |
| 倒地 | 3 + 静止 | 4 |
| 搬运伤员 | 4–6 | 4 |
| 死亡/转化 | 4–6 | 4 |

帧率建议 8–12 fps，不用过多补间破坏像素质感。

### 16.3 轮廓可读性

- 武器类别和背包大小在普通缩放下可辨；
- 友方、陌生人、敌对者不只靠衣服颜色区分；
- 感染者保留人形与生前身份痕迹，不做夸张怪兽；
- 倒地者与尸体姿态明显不同；
- 血迹和感染覆盖不遮住阵营/选择轮廓。

## 17. 环境素材包

### 17.1 通用核心包

- 道路、人行道、路缘、裂缝、积水；
- 常见墙地、门窗、楼梯、围栏；
- 床、桌椅、柜台、货架、容器；
- 医疗、学校、办公、仓储和维修道具；
- 植被、水域、垃圾、废弃车辆；
- 路障、临时床铺、隔离和求救标记；
- 火焰、烟雾、供电、潮湿和污染状态。

### 17.2 地区包覆盖

地区包替换建筑立面、道路标识、常见家具、食物外观和公共设施细节，不复制通用碰撞与逻辑素材。地区差异优先放在首屏可见的街道、屋顶和标志结构。

### 17.3 地点故事层

每种 POI 至少有三组可叠加状态素材：

- 静默日前的正常用途；
- 临时避难/封锁；
- 搜刮、占用、火灾或感染后的状态。

## 18. 光照与特效

- 白天室内有环境光，不让未开灯区域完全不可见；
- 手电是方向性信息工具，不是唯一可玩性开关；
- 基地暖光只影响局部，保留建筑结构可读；
- 火焰、枪口焰和警报闪烁提供“降低/关闭闪烁”版本；
- 雨、烟和雾限制透明层数量，避免遮挡单位轮廓；
- 镜头震动默认克制，枪械和爆炸强度可单独调节；
- 伤害反馈使用短时轮廓、音效和角色动作，不用全屏长时间变红。

## 19. UI 视觉规范

- 正文使用清晰的无衬线字体，不用像素字体承载长文本；
- 像素字体只用于短标题、计数或世界内显示；
- 字距保持 0，不使用负字距；
- 普通面板圆角不超过 4–6 px，保持工具感；
- 页面分区优先使用留白、细线和背景明度，不把每块内容都做成卡片；
- 不在卡片内再嵌套卡片；
- 图标按钮有悬停工具提示和键位；
- 文本按钮只用于明确命令，如“开始行动”“结束今日”；
- 二元设置使用复选框/开关，模式使用分段控件，数值使用滑杆或步进器；
- 危险确认按钮写清行动，如“遗弃车辆并徒步”，不用泛化“确定”。

## 20. 音频方向

### 20.1 音频层级

1. 战术信息：抓取、破门、枪声、空仓、伤员呼救；
2. 环境信息：远处感染声、风雨、发电机、广播；
3. 角色反馈：短句、喘息、争执、安慰；
4. UI 反馈：选择、警告、完成、错误；
5. 音乐：低频出现，支持情绪而不持续占满。

战术信息拥有最高混音优先级。音乐和环境不能盖住可影响决策的声音。

### 20.2 环境音景

城市音景由远景、中景、近景三层组成：

- 远景：风、远方火灾、偶发广播、城市空腔；
- 中景：雨打材质、松动招牌、远处碰撞、线路嗡鸣；
- 近景：脚步、衣物、背包、门锁、容器和呼吸。

不同 POI 使用材质混响和局部声源区别，不靠持续惊吓音效。

### 20.3 感染者声音

- 不同警觉阶段有可分辨但克制的声线；
- 躁行者冲刺前有极短、可反应的呼吸/碰撞提示；
- 伏听者在细心停留时能听到环境线索；
- 屏外声音同时提供视觉方向辅助；
- 同屏大量感染者使用聚合和变体，避免音量叠加失控。

### 20.4 音乐

- 基地安全、计划压力、未知探索、警戒升级、失去角色、迁徙分别使用可叠层主题；
- 普通探索允许长时间无音乐，让环境声承担紧张；
- 战斗音乐不应每见一个游荡者就启动；
- 角色死亡后避免立即播放夸张悲壮段落，给环境和反应留空间；
- 主界面主题可复用“无线电坐标”动机，建立品牌声音。

### 20.5 格式

- 源文件：48 kHz/24-bit WAV；
- 游戏内短音效：按引擎推荐无损/轻压缩；
- 长环境与音乐：OGG 流式播放；
- 循环点在源文件中记录；
- 文件名使用 ASCII ID，不把显示文本写入文件名。

## 21. 文本与本地化

- 逻辑数据只引用本地化键；
- 中文不依赖空格换行，英文允许长度增加 30%–50%；
- 按钮设置最小与最大宽度，长词自动换行或使用可访问缩放；
- 角色代词、称谓和姓名顺序由语言包处理；
- 数字、日期、时间和单位使用本地化格式；
- 广播必须有字幕和历史文本；
- 地图原始地名可保留本地文字，并提供可选转写/翻译字段；
- 缺失翻译显示稳定的后备语言，不显示键名给正式玩家。

## 22. 素材命名

```text
角色：chr_<部位>_<款式>_<颜色>_<动作>_<方向>.png
环境：env_<地区>_<poi>_<物件>_<状态>.png
地块：tile_<地区>_<材质>_<变体>.png
图标：ico_<系统>_<含义>_<尺寸>.png
特效：vfx_<类型>_<强度>.png
音效：sfx_<系统>_<动作>_<材质>_<变体>.wav
环境：amb_<地区>_<地点>_<天气>_<时段>.ogg
音乐：mus_<章节>_<情绪>_<层>.ogg
```

文件名只用小写 ASCII、数字和下划线。显示名称在数据表和本地化文件中维护。

## 24. 视觉与交互验收

至少检查 1280×720、1920×1080、2560×1440、16:10 和超宽屏：

- 任何常驻 UI 不遮挡关键战术区域和撤退入口；
- 最长中文/英文按钮文本不溢出；
- UI 160% 时仍可访问所有命令，必要区域可滚动；
- 角色条、地图筛选和事件选项不会因动态内容改变核心布局尺寸；
- 色觉模拟下阵营、警戒、资源和路径仍能凭图标/线型区分；
- 像素素材无模糊采样、接缝和非整数抖动；
- 四人同时战斗时声音、轮廓和状态均可识别；
- 黑暗、雨和烟叠加后，已知单位仍符合最低可见性；
- 所有图标按钮有工具提示，所有不可用命令有原因；
- 键盘重绑、字幕、闪烁和镜头设置即时生效并保存。
